Convey Images

A self-hosted, privacy-respecting image paste board for Convey chat — strip metadata, tokenize URLs, and share images safely.

Features

Convey Images is the companion image hosting service for the Convey chat client. It strips sensitive metadata, generates anonymous URLs, and keeps your image sharing private — whether you're in a regular IRC channel or a secure encrypted channel.

🔎

EXIF Stripping

Every uploaded image is re-encoded from scratch, removing all EXIF metadata — GPS coordinates, camera info, timestamps, ICC profiles, and any other embedded data.

🔗

Tokenized URLs

Original filenames are never exposed. Each image gets a random 12-character token as its URL, preventing filename-based tracking or information leakage.

📈

Duplicate Detection

Content-hash (SHA-256) based deduplication detects identical uploads and returns the existing URL — saving storage without re-uploading.

📷

Image Resizing

Oversized images are automatically downscaled to a configurable maximum dimension (default 4096px) using high-quality Lanczos3 resampling.

🖥

Web Upload Interface

A built-in drag & drop web interface with clipboard paste support. No client needed — just visit the service URL in any browser and drop your image.

⚡

REST API

Simple multipart POST endpoint for programmatic uploads. Integrated directly into the Convey chat client for seamless in-chat image sharing.

🖼

Gallery View

Browse the 50 most recent uploads at the /gallery endpoint. Quick visual overview of what's been shared.

🕐

Auto Cleanup

Configurable retention period and maximum image count. Old images are automatically purged on an hourly cycle, or the oldest images are removed when limits are reached.

🌐

OG Meta Tags

View pages at /v/ include Open Graph and Twitter Card meta tags, so shared links get rich image previews in chat clients and social platforms.

🔒

Delete Tokens

Every upload returns a unique delete URL. Only the uploader can remove their image — no accounts or authentication required.

🚀

Built with Rust & Axum

Async from top to bottom with Tokio and Axum. Minimal resource footprint, fast image processing, and rock-solid reliability.

📦

Self-Hostable

Single static binary with TOML configuration. Deploy anywhere — your server, your rules, your data. Full Nginx and Apache proxy configs included.

API Reference

Convey Images exposes a straightforward HTTP API for uploading, viewing, and managing images. All responses are JSON unless otherwise noted.

POST

/upload

Upload an image via multipart form data. The file field should be named file or image. Returns a JSON response with the view URL, direct URL, delete URL, dimensions, and file size.

GET

/i/{token}.{ext}

Serve the raw image with appropriate Content-Type, immutable caching headers, and X-Content-Type-Options: nosniff.

GET

/v/{token}.{ext}

HTML view page with Open Graph meta tags for link previews. Displays the image in a styled page suitable for sharing.

GET

/api/info/{token}

Returns JSON metadata about the image: token, extension, content type, dimensions, file size, upload timestamp, and content hash.

POST

/delete/{delete_token}

Delete an image using the unique delete token returned at upload time. Removes both the image file and its metadata.

GET

/gallery

Browse the 50 most recently uploaded images in a visual gallery page. Sorted newest-first.

Deployment

Convey Images ships as a single binary. Build it, configure it, and run it behind your reverse proxy of choice.

1

Build

Compile with Cargo:
cargo build --release -p convey-images
Or cross-compile for Linux with musl for a fully static binary.

2

Configure

Copy convey.toml.example to /etc/convey-images/config.toml and set your bind address, base URL, storage path, and limits.

3

Run

Start the service:
convey-images -c /etc/convey-images/config.toml
A systemd unit file is included for production use.

4

Reverse Proxy

Point Nginx or Apache at the bind address. Sample configs for both are included. Add TLS with certbot.

Configuration

All settings are controlled via a single TOML file. Here's every option:

Setting Default Description
bind_address 0.0.0.0:8844 HTTP server bind address
base_url https://convey.pugbot.net Public URL used for generating image links
storage_dir /var/lib/convey-images Directory where images and metadata are stored
max_upload_size 20 MB Maximum file size per upload (in bytes)
max_images 50,000 Maximum number of stored images (oldest evicted when exceeded)
retention_days 0 (forever) Auto-delete images older than N days (0 disables)
max_dimension 4096 Maximum width or height — larger images are downscaled

Frequently Asked Questions

Is Convey Images integrated into the Convey chat client?

Yes. When you paste or attach an image in Convey, the client uploads it to your configured Convey Images instance via the POST /upload API and automatically shares the resulting URL in chat.

What image formats are supported?

JPEG, PNG, GIF, and WebP are all accepted. GIF and WebP images are re-encoded as PNG to ensure complete metadata stripping. JPEG images are re-encoded as JPEG.

Does it really remove all EXIF data?

Yes. Images are fully decoded and then re-encoded from the raw pixel data. This removes all embedded metadata — EXIF, GPS coordinates, camera make/model, timestamps, ICC color profiles, and any other chunks. Nothing survives re-encoding.

What happens when I upload a duplicate image?

Convey Images computes a SHA-256 hash of the processed image. If an identical image already exists, it returns the existing URL instead of creating a new entry — saving storage space and bandwidth.

How does auto-cleanup work?

If retention_days is set (non-zero), a background task runs every hour and deletes images older than the configured retention period. Additionally, when max_images is reached, the oldest image is removed to make room for new uploads.

Do I need an account to use it?

No. Convey Images requires no authentication for uploading or viewing. Each upload returns a unique delete token that only the uploader receives — that's the only way to remove an image.

Can I use it without the Convey chat client?

Absolutely. The web interface supports drag & drop, file picker, and clipboard paste. You can also use any HTTP client or curl to upload via the API:
curl -F "file=@photo.jpg" https://convey.pugbot.net/upload

Is it open source?

Yes. Convey Images is part of the NAIS project, licensed under Apache 2.0. The full source is in the convey/ directory.
